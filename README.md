# AI 教育图谱 (AI Education Roadmap)

> 让每个人获得顶级教育资源

## 项目愿景

本项目致力于推动**优质教育普惠每个人** — 让普通人即可获得以往只有顶尖教育资源才能提供的学习指导和能力培养方案。核心交付物为结构化知识内容，用户可直接阅读获取指导，也可结合 AI 工具获得个性化建议。

我们相信：一个人的成长机会不应被出生地、家庭背景或经济条件所限制。

## 核心理念

- **全人发展**：认知心理、身体能力、父母支持三条主线并行，身心一体
- **科学依据**：所有能力划分和培养方案均基于发展心理学、运动科学等成熟理论
- **知识驱动**：核心交付物为结构化知识内容，直接可用；AI 作为可选的个性化辅助工具
- **开源协作**：社区共建，分层治理，权威可追溯
- **终身学习**：覆盖 0 岁到老年全年龄层，学习没有终点

## 教育图谱总览

项目按人生阶段组织能力培养路径，每个阶段包含**认知与心理**和**身体能力**双主线，家庭期额外包含**父母支持**主线：

| 阶段 | 年龄/阶段 | 认知与心理主线 | 身体能力主线 | 父母支持主线 |
|------|----------|-------------|------------|------------|
| 青春期 | 14-18 | 形式运算思维、身份探索、认识论思考、亲密关系能力... | PHV身高突增期、肌肉系统训练、心肺机能提升 | |
| 大学期 | 18-22 | 后形式思维、专业体系构建、财务素养入门、自主学习... | 生理机能峰值期、运动习惯终身锚定、睡眠优化 | |
| 职场开始 | 22-28 | 专业深化、财务素养进阶、职业身份确立... | 系统化训练方法、运动习惯巩固、久坐对抗 | |
| 职场发展 | 28-40 | 专业精通、领导力发展、压力管理、意义感... | 体能维持、久坐对抗、核心力量、代谢率下降应对 | |

## 双线结构

### 主线：阶段 → 必备技能

按人生阶段划分，每个阶段列出该阶段**必须培养的核心能力**，所有人均应关注。

### 副线：兴趣 → 跨阶段学习路径

按兴趣领域划分（登山、语言、植物学、动物学等），每个兴趣领域的内容按**学习路线**（入门→进阶→高阶）组织在 `interests/` 目录中。兴趣是跨阶段的独立学习路径——一个人学登山，从入门到进阶是一条完整的纵向路径，不因人生阶段变化而中断。

阶段目录通过链接引用适合该阶段的兴趣内容。

用户可从任一入口进入：
- 从阶段入手 → 获取该阶段所有核心能力 + 推荐兴趣链接
- 从兴趣入手 → 获取该兴趣的完整学习路线，从当前水平进入

### 目录结构规范

1. **阶段目录命名**：统一使用中文目录名
   - 14岁后（面向学习者本人）：`青春期`、`大学期`、`职场开始`、`职场发展`、`家庭期`、`中年期`、`老年期`
   - 0-18岁（面向家长）：归属 `家庭期/育儿指导/` 下，按年龄分目录：`0-3`、`3-6`、`6-9`、`9-12`、`12-14`、`14-18`
2. **子目录命名**：根据阶段特点灵活命名，使用中文
   - 0-18岁育儿指导：保持 `认知与心理/` + `身体能力/` 的结构化划分
   - 14岁后学习者目录：按阶段关注点命名，如 `身份探索/`、`职业发展/`、`健康管理/` 等
3. **兴趣目录**：`interests/` 按兴趣主题组织，内部按学习路线（入门→进阶→高阶）存放内容；阶段目录通过链接引用适合该阶段的兴趣
4. **主线内容存放在 `stages/` 目录中**，兴趣内容存放在 `interests/` 目录中
5. 每个阶段目录必须包含 `_index.md` 作为阶段入口

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 唯一标识，格式：`阶段-主线-主题-序号`，如 `0-3-core-attachment-01` |
| `stage` | 是 | 所属阶段，与目录名一致 |
| `track` | 是 | `core`（主线必备技能）或 `interests`（副线兴趣） |
| `domain` | 是 | 能力领域：`cognitive-psychological`、`physical`、或兴趣名称 |
| `topic` | 是 | 主题名称，英文短横线连接 |
| `age_range` | 是 | 精确年龄范围，格式：`0-18m`（月）、`3-6y`（年）、`adult` |
| `difficulty` | 是 | `foundational`（入门）、`intermediate`（进阶）、`advanced`（高阶） |
| `review_status` | 是 | `expert-reviewed`（专家审核）、`community-contributed`（社区贡献）、`draft`（草稿） |
| `reviewers` | 否 | 专家审核者姓名列表 |
| `last_reviewed` | 否 | 最后审核日期，格式 YYYY-MM-DD |
| `references` | 是 | 引用文献列表，APA 格式，须在 `references/` 目录中有记录 |
| `tags` | 是 | 标签列表，便于搜索和索引 |
| `learning_paths` | 否 | 所属学习路径的 id 列表 |
| `related_prompts` | 否 | 关联内容的 id 列表 |

### 内容编写规范

1. **直接可用**：内容应直接可读，不需要依赖 AI 工具
2. **通俗易懂**：用家长/学习者能理解的语言，避免过多学术术语
3. **实操导向**：每个主题包含"可以做的事"和"今天就能做的一件事"
4. **问题驱动**：每个主题包含"常见问题"直接回答高频疑问
5. **语言**：中文为主，术语保留英文原文（如"执行功能 Executive Function"）
6. **权威引用**：所有学习建议必须有理论或实证依据，禁止凭空编造
7. **无商业内容**：不得包含任何商业推广或品牌推荐

## 学习路径文件规范

学习路径定义了内容之间的先后关系、完成标准和触发条件。

### 文件位置

```
paths/
├── _index.md                          # 路径总览
├── <阶段>/
│   └── <领域>.md                      # 该阶段该领域的学习路径
└── <主题>/
    └── <子主题>.md                    # 跨阶段主题路径
```

### 文件格式

```markdown
---
id: <路径标识，如 0-3-cognitive-psychological>
name: <路径名称>
stage: <所属阶段>
domain: <能力领域>
description: <一句话描述>
estimated_duration: <预计时长>
review_status: expert-reviewed | community-contributed | draft
---

# <路径名称>

## 路径总览

<简要说明本路径覆盖的能力范围和设计逻辑>

## 步骤

### Step 1: <步骤名称>

- **content_id**: <内容 id>
- **prerequisites**: [<前置内容 id 列表，空列表表示起始步骤>]
- **age_range**: <适用年龄范围>
- **completion_criteria**:
  - <完成标准 1>
  - <完成标准 2>
- **trigger**:
  - condition: user-initiated | time-based | state-based | event-based
  - description: <触发条件的自然语言描述>
  - agent_should: <Agent 收到触发信号后的具体行动指引>
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `content_id` | 是 | 对应的内容 id |
| `prerequisites` | 是 | 前置步骤的内容 id 列表，空列表 `[]` 表示起始步骤 |
| `age_range` | 是 | 该步骤适用的年龄范围 |
| `completion_criteria` | 是 | 自然语言描述的完成标准，Agent 据此判断用户是否掌握 |
| `trigger.condition` | 是 | 触发类型：`user-initiated`（用户主动）、`time-based`（定时）、`state-based`（状态触发）、`event-based`（事件触发） |
| `trigger.description` | 是 | 触发条件的自然语言描述 |
| `trigger.agent_should` | 是 | Agent 的行动指引 |

### 设计原则

1. **路径 = 学习曲线**：一条路径覆盖一个能力领域从入门到掌握的完整过程
2. **步骤有先后**：prerequisites 定义前置依赖，Agent 据此判断用户是否"准备好"
3. **完成可评估**：completion_criteria 用自然语言描述，Agent 在对话中判断
4. **触发可自动化**：trigger 告诉 Agent 何时主动推送，而非等用户来问
5. **路径独立于内容**：路径文件单独存放，路径变化不需要修改内容文件

## 内容治理

| 层级 | 标识 | 说明 |
|------|------|------|
| 专家审核 | `review_status: expert-reviewed` | 核心图谱和基础内容，由教育/心理学/运动科学领域专家审核 |
| 社区贡献 | `review_status: community-contributed` | 社区成员贡献的扩展内容，经过至少一位社区成员评审 |
| 草稿 | `review_status: draft` | 初稿，尚未经过审核，仅供参考 |

**类型（type）**：

| 类型 | 说明 |
|------|------|
| `content` | 新增或修改内容 |
| `index` | 新增或修改阶段/兴趣索引文件 |
| `refs` | 新增或修改参考文献 |
| `roadmap` | 修改教育图谱框架 |
| `docs` | 修改项目文档（README、CONTRIBUTING 等） |
| `fix` | 修复内容错误（事实错误、引用错误等） |
| `i18n` | 国际化翻译 |

**范围（scope）**：阶段或目录名，如 `0-3`、`university`、`family`、`interests`、`refs`

**示例**：
```
content(0-3): 新增安全依恋培养内容
fix(12-15): 修正 PHV 年龄范围描述
refs: 新增 Ayres 感觉统合理论原文引用
index(interests): 新增登山兴趣跨年龄索引
roadmap: 调整学龄中阶段能力划分
```

### Pull Request 规范

1. **一个 PR 做一件事**：不要混合不同阶段或不同类型的内容变更
2. **标题格式**：`[type] 简短描述`，如 `[content] 新增 0-3 岁安全依恋培养内容`
3. **描述内容**：
   - 变更说明
   - 引用的理论依据（如新增内容）
   - 是否需要专家审核
4. **自检清单**：
   - [ ] 内容文件遵循格式规范
   - [ ] 引用文献在 `references/` 中有记录
   - [ ] `_index.md` 已同步更新（如涉及新主题）
   - [ ] `related_prompts` 中的关联内容已存在
   - [ ] `review_status` 已正确标注

### 新增内容流程

1. Fork 仓库 → 创建分支
2. 按规范创建内容文件
3. 更新对应阶段的 `_index.md`
4. 如涉及新兴趣领域，在 `interests/` 下创建索引
5. 如引用新文献，在 `references/` 中添加完整记录
6. 提交 PR，等待审核
