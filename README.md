# AI 教育图谱 (AI Education Roadmap)

> 让每个人借助 AI 获得顶级教育资源

## 项目愿景

随着 AI 能力的不断增强，它在教育领域的潜力远未被充分释放。本项目致力于推动**教育平权** — 让普通人借助 AI 即可获得以往只有顶尖教育资源才能提供的学习指导和能力培养方案。

我们相信：一个人的成长机会不应被出生地、家庭背景或经济条件所限制。

## 核心理念

- **全人发展**：认知心理与身体能力双主线并行，身心一体
- **科学依据**：所有能力划分和培养方案均基于发展心理学、运动科学等成熟理论
- **模型无关**：核心交付物为结构化 Prompt，兼容任何 AI 模型（Claude、ChatGPT、Gemini 等）
- **开源协作**：社区共建，分层治理，权威可追溯
- **终身学习**：覆盖 0 岁到老年全年龄层，学习没有终点

## 教育图谱总览

项目按人生阶段组织能力培养路径，每个阶段包含**认知与心理**和**身体能力**双主线：

| 阶段 | 年龄 | 认知与心理主线 | 身体能力主线 |
|------|------|-------------|------------|
| 婴幼儿 | 0-3 | 客体永久性、因果关系推理、联合注意、安全依恋、自我意识觉醒... | 粗大动作链、精细动作链、感觉统合、口部运动 |
| 学龄前 | 3-6 | 心理理论、执行功能、延迟满足、情绪调节策略... | 基本动作技能、平衡与协调、手眼协调、利手确立 |
| 学龄初 | 6-9 | 守恒与逻辑运算、阅读习得、元认知萌芽、友谊技能... | 过渡性动作技能、游泳/骑行/球类入门、体态养成 |
| 学龄中 | 9-12 | 抽象思维萌芽、自我调节学习、批判性思维种子、抗逆力... | 专项运动启蒙、速度与敏捷性、运动习惯锚定 |
| 青春期前 | 12-15 | 形式运算思维、身份探索、情绪波动、同伴影响峰值... | PHV身高突增期、力量/速度训练窗口、损伤预防 |
| 青春期 | 15-18 | 认识论思考、身份整合、亲密关系能力、家庭关系重构... | 肌肉系统训练、心肺机能提升、训练系统化 |
| 青年初 | 18-25 | 后形式思维、专业体系构建、财务素养、亲密关系建立... | 生理机能峰值期、运动习惯终身锚定、睡眠优化 |
| 成年 | 25-40 | 专业精通、领导力发展、压力管理、意义感... | 体能维持、久坐对抗、核心力量、代谢率下降应对 |
| 中年 | 40-60 | 晶体智力峰值、智慧与判断力、知识传承... | 肌少症预防、骨密度维护、心血管风险应对 |
| 长者 | 60+ | 智慧结晶、认知衰退预防、生命叙事整合... | 防跌倒训练、功能独立性维护、温和有氧运动 |

详细框架见 [roadmap.md](roadmap.md)。

## 双线结构

### 主线：年龄 → 必备技能

按人生阶段划分，每个阶段列出该年龄段**必须培养的核心能力**，所有人均应关注。

### 副线：兴趣 → 跨年龄路径

按兴趣领域划分（登山、语言、植物学、动物学等），每个兴趣领域内按年龄阶段提供对应的 Prompt。兴趣目录本身只存放**索引文件**，实际 Prompt 存放在对应的 `stages/` 目录中。

用户可从任一入口进入：
- 从年龄入手 → 获取该阶段所有核心能力 + 可选兴趣 Prompt
- 从兴趣入手 → 获取该兴趣在各年龄阶段的 Prompt 索引

### 目录结构规范

1. **年龄阶段目录**命名格式：`0-3`、`3-6`、`6-9`、`9-12`、`12-15`、`15-18`、`18-25`、`25-40`、`40-60`、`60+`（使用连字符连接，`60+` 表示 60 岁以上）
2. **主线目录**固定为 `core/`，下分 `cognitive-psychological/` 和 `physical/` 两个子目录
3. **兴趣目录**固定为 `interests/`，按兴趣名称创建子目录
4. **Prompt 文件只存在于 `stages/` 目录中**，`interests/` 目录仅存放索引文件
5. 每个阶段目录必须包含 `_index.md` 作为阶段入口

## Prompt 文件格式规范

每个 Prompt 文件必须遵循以下格式：

```markdown
---
id: <阶段>-<主线>-<主题>-<序号>
stage: <年龄阶段>
track: core | interests
domain: cognitive-psychological | physical | <兴趣名称>
topic: <主题名称>
age_range: <精确年龄范围，如 0-18m、6-9y>
difficulty: foundational | intermediate | advanced
review_status: expert-reviewed | community-contributed | draft
reviewers: [<审核者姓名>]
last_reviewed: <YYYY-MM-DD>
references:
  - <APA格式引用>
tags: [<标签1>, <标签2>]
related_prompts: [<关联prompt的id>]
---

# <主题名称>

## 学习目标

<1-2句话描述本 Prompt 的学习目标>

## 为什么重要

<简要说明该能力在此阶段的重要性，引用理论依据>

## Prompt（单轮模式）

复制以下内容发送给 AI，获得完整的学习指导和活动方案：

> 你是一位<角色定义>。请为<目标用户>提供一份关于「<主题>」的详细指导方案。要求：<具体要求列表>。请用通俗易懂的语言，适合非专业家长/学习者阅读。

## Prompt（多轮互动模式）

复制以下内容发送给 AI，进入对话式学习：

> 你是一位<角色定义>，我将向你咨询关于<主题>的问题。我的<具体情况占位符>。请先了解我的具体情况，然后像一位耐心的老师一样，通过提问和讨论引导我理解<主题>的要点，每次只讲一个要点，确认我理解后再继续。不要一次性给出所有信息。

## 延伸探索

- [<关联主题1>](<相对路径>) — <一句话说明关联>
- [<关联主题2>](<相对路径>) — <一句话说明关联>
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 唯一标识，格式：`阶段-主线-主题-序号`，如 `0-3-core-attachment-01` |
| `stage` | 是 | 所属年龄阶段，与目录名一致 |
| `track` | 是 | `core`（主线必备技能）或 `interests`（副线兴趣） |
| `domain` | 是 | 能力领域：`cognitive-psychological`、`physical`、或兴趣名称 |
| `topic` | 是 | 主题名称，英文短横线连接 |
| `age_range` | 是 | 精确年龄范围，格式：`0-18m`（月）、`3-6y`（年）、`adult` |
| `difficulty` | 是 | `foundational`（入门）、`intermediate`（进阶）、`advanced`（高阶） |
| `review_status` | 是 | `expert-reviewed`（专家审核）、`community-contributed`（社区贡献）、`draft`（草稿） |
| `reviewers` | 否 | 专家审核者姓名列表 |
| `last_reviewed` | 否 | 最后审核日期，格式 YYYY-MM-DD |
| `references` | 是 | 引用文献列表，APA 格式，须在 `references/` 目录中有记录 |
| `tags` | 是 | 标签列表，便于搜索和索引 |
| `related_prompts` | 否 | 关联 Prompt 的 id 列表 |

### Prompt 编写规范

1. **模型无关**：不引用特定 AI 模型的功能或特性，确保在任何主流 AI 对话中均可使用
2. **占位符**：使用 `[填写：xxx]` 标记用户需要自定义的部分
3. **单轮模式**：面向希望快速获取完整方案的用户，一次性输出
4. **多轮模式**：面向希望深度学习的用户，AI 引导式对话
5. **语言**：中文为主，术语保留英文原文（如"执行功能 Executive Function"）
6. **权威引用**：所有学习建议必须有理论或实证依据，禁止凭空编造
7. **无商业内容**：Prompt 中不得包含任何商业推广或品牌推荐

## 内容治理

| 层级 | 标识 | 说明 |
|------|------|------|
| 专家审核 | `review_status: expert-reviewed` | 核心图谱和基础 Prompt，由教育/心理学/运动科学领域专家审核 |
| 社区贡献 | `review_status: community-contributed` | 社区成员贡献的扩展内容，经过至少一位社区成员评审 |
| 草稿 | `review_status: draft` | 初稿，尚未经过审核，仅供参考 |

## 提交规范

### Commit 消息格式

```
<type>(<scope>): <description>

<optional body>
```

**类型（type）**：

| 类型 | 说明 |
|------|------|
| `content` | 新增或修改 Prompt 内容 |
| `index` | 新增或修改阶段/兴趣索引文件 |
| `refs` | 新增或修改参考文献 |
| `roadmap` | 修改教育图谱框架 |
| `docs` | 修改项目文档（README、CONTRIBUTING 等） |
| `fix` | 修复内容错误（事实错误、引用错误等） |
| `i18n` | 国际化翻译 |

**范围（scope）**：年龄阶段或目录名，如 `0-3`、`interests`、`refs`

**示例**：
```
content(0-3): 新增安全依恋培养 Prompt
fix(12-15): 修正 PHV 年龄范围描述
refs: 新增 Ayres 感觉统合理论原文引用
index(interests): 新增登山兴趣跨年龄索引
roadmap: 调整学龄中阶段能力划分
```

### Pull Request 规范

1. **一个 PR 做一件事**：不要混合不同阶段或不同类型的内容变更
2. **标题格式**：`[type] 简短描述`，如 `[content] 新增 0-3 岁安全依恋培养 Prompt`
3. **描述内容**：
   - 变更说明
   - 引用的理论依据（如新增内容）
   - 是否需要专家审核
4. **自检清单**：
   - [ ] Prompt 文件遵循格式规范
   - [ ] 引用文献在 `references/` 中有记录
   - [ ] `_index.md` 已同步更新（如涉及新主题）
   - [ ] `related_prompts` 中的关联 Prompt 已存在
   - [ ] `review_status` 已正确标注

### 新增内容流程

1. Fork 仓库 → 创建分支
2. 按规范创建 Prompt 文件
3. 更新对应阶段的 `_index.md`
4. 如涉及新兴趣领域，在 `interests/` 下创建索引
5. 如引用新文献，在 `references/` 中添加完整记录
6. 提交 PR，等待审核
