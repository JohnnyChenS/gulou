# AI 教育图谱项目

## 项目概述

全人生阶段的学习、成长、能力培养、兴趣培养平台。以开源知识内容为核心，为用户提供结构化的成长指导。覆盖 0 岁到老年全年龄层，育儿是当前切入点，最终目标是回归个人成长和兴趣学习。用户可直接阅读获取指导，也可结合 AI 工具获得个性化建议。

## 核心设计决策

- **三条主线**：认知心理 + 身体能力 + 父母支持（父母支持仅在家庭期）
- **双入口**：主线（阶段→必备技能）、副线（兴趣→跨阶段路径）
- **用户分层**：0-18岁面向家长（育儿指导），14-18岁同时面向家长（育儿指导）和学习者本人（青春期阶段），18岁后完全过渡到学习者本人
- **阶段划分**：14岁后按生活阶段（青春期、大学期、职场开始、职场发展、家庭期、中年期、老年期），0-18岁育儿指导内容归属家庭期/育儿指导
- **灵活目录结构**：0-18岁育儿指导保持结构化（认知与心理、身体能力），14岁后学习者目录按阶段特点灵活命名（如身份探索、职业发展、健康管理等）
- **运动归属**：0-14岁在身体能力目录，14-18岁在身体发育目录，18岁后运动归入兴趣目录，但中年期/老年期保留健康维护目录中的必需运动
- **内容归属**：主线内容在 `stages/`，兴趣内容在 `interests/`（按学习路线组织，阶段目录通过链接引用）
- **分层治理**：核心内容专家审核，社区贡献标记"community-contributed"
- **知识驱动**：核心交付物为结构化知识内容，AI 作为可选的个性化辅助工具
- **中文优先**，后续国际化
- **纯内容仓库**：Markdown 文件，无代码工具
- **父母支持维度**：不仅关注孩子能力发展，也关注照料者的心理状态和育儿指导（集中在家庭期）

## 产品策略

### 核心问题

学习乐器、语言、运动，或育儿中的习惯培养、心理健康建设，都是漫长的过程，需要持续引导和动态调整。

### 解决方向：知识驱动 + AI 辅助

核心交付物为结构化知识内容——涵盖发展里程碑、实操指南、常见问题解答和权威文献支撑。用户可直接阅读获取指导，也可结合 AI 工具获得个性化建议。
- 知识内容作为核心价值，直接可用
- AI 作为可选的个性化辅助工具
- 内容按人生阶段和主题组织，便于查找

### 工具层设计：多个垂直小工具

底层共享同一个开源知识库，上层拆分为多个独立垂直工具，各自独立上线、验证需求、变现：
- 育儿工具（输入月龄 → 成长指导方案）
- 兴趣学习工具（选兴趣 → 跨阶段学习路径）
- 习惯养成工具（输入习惯 → 基于行为科学的培养计划）
- 职业发展工具（输入职业阶段 → 能力提升方案）
- 健康工具（输入年龄/目标 → 运动/营养/心理方案）

### 变现路径

- **短期**：开源迭代内容 + GitHub Sponsors / 爱发电
- **中期**：垂直小工具上线，免费限次 + 付费无限
- **长期**：培训/咨询/企业合作

## 目录结构

```
stages/<阶段>/_index.md                      # 阶段概述+能力清单+学习路径
stages/<阶段>/<主题目录>/                     # 按阶段特点命名的主题目录
stages/家庭期（25-45岁）/育儿指导/                       # 父母支持主线（含0-18岁育儿指导）
stages/家庭期（25-45岁）/育儿指导/<年龄>/认知与心理/     # 0-18岁认知发展
stages/家庭期（25-45岁）/育儿指导/<年龄>/身体能力/       # 0-18岁运动发育
interests/<兴趣名>/                          # 兴趣领域（独立学习路径）
interests/language/                        # 语言学习（母语发展 + 英语学习）
interests/<兴趣名>/_index.md                 # 学习路线总览+阶段映射
interests/<兴趣名>/beginner.md               # 入门内容
interests/<兴趣名>/intermediate.md           # 进阶内容
interests/<兴趣名>/advanced.md               # 高阶内容
references/                                  # 权威文献库
```

**阶段划分：**
- 14岁后（面向学习者本人）：青春期（14-18）、大学期（18-22）、职场开始（22-28）、职场发展（28-40）、家庭期（25-45）、中年期（40-60）、老年期（60+）
- 0-18岁（面向家长）：归属家庭期/育儿指导/，按年龄分：0-3、3-6、6-9、9-12、12-14、14-18

## 内容文件格式

### 核心知识文件

每个内容文件必须包含：
- frontmatter：id、stage、track、domain、topic、age_range、difficulty、review_status、references、tags、related_prompts
- 一句话总结：3 秒抓住重点
- 你需要知道的：核心知识和理论依据的通俗解释
- 各月龄/年龄段该关注什么：按时间拆分的发展里程碑
- 可以做的事：3-5 个具体活动（含做法、原因、频率）
- 常见问题：5-8 个高频问题的直接回答
- 需要警惕的信号：什么情况该看医生/求助专业人士
- 今天就能做的一件事：具体到动作级别的即时行动
- 延伸探索：关联其他主题

### 执行日历（可选）

在核心知识内容之后、延伸探索之前，可追加执行日历，解决"今天打开干什么"的问题：
- 按周/阶段拆分的逐日任务表
- 可包含分龄/分用户版本（如家长辅导版 + 成人自学版）
- 每段附阶段自检清单
- 建议包含容错机制（"错过一天不用补"）
- 顶部可加行为设计建议（提醒设置、最小执行方案）
- 任务可与上文知识内容交叉引用

示例见 `interests/language/英语学习/beginner.md` 的 30 周执行日历。

### 学习路径文件

存放在 `paths/` 目录，定义内容之间的先后关系、完成标准和触发条件。

完整规范见 README.md。
