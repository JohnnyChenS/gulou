# 学习路径索引

学习路径定义了知识内容之间的先后关系、完成标准和触发条件。每条路径对应一个能力领域从入门到掌握的完整过程。

## 两层设计

```
学习路径
├── 知识路径（Knowledge Path）   →  学习顺序 + 前置依赖 + 触发条件
└── 执行路线（Execution Route）  →  每日任务 + 阶段自检 + 容错机制
```

- **知识路径**：告诉 AI Agent 和用户在学什么、先学什么后学什么，定义内容之间的前置依赖
- **执行路线**：告诉用户"今天打开做什么"，将知识转化为可执行的任务表

> 已创建执行路线的领域，在路径末尾会有 `→ [执行路线](...)` 链接。

```

paths/<年龄段>/physical.md                知识路径（身体能力）
paths/<兴趣名>/<方向>.md                   兴趣学习路径
paths/parenting/<名称>.md                 父母支持路径
```

## 路径文件格式规范

每个路径文件包含：
- **frontmatter**：id、name、stage、domain、description、estimated_duration
- **路径总览**：本路径覆盖的能力范围和设计逻辑
- **步骤列表**：每个步骤包含 prompt、prerequisites、age_range、completion_criteria、trigger
- **执行路线（可选）**：链接到对应的执行日历文件
